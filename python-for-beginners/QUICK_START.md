# Quick Start

**Your daily schedule: 1.5 to 2 hours per day, 6 days per week.**

Set aside the same time slot every day — morning before school, evening after dinner,
whatever fits your life. The time of day matters less than the consistency.
Short daily sessions beat long weekend marathons every time. See `LEARNING_APPROACH.md`
for the science behind this.

---

## Finding where you left off

1. Open `COURSE_CATALOG.md`.
2. Find the last module with a `☑` checkbox.
3. The next `☐` module is where you continue.
4. Inside that module's folder, open `README.md` and check which exercises you have
   not completed yet.

---

## Resuming a module mid-way

Each module's `README.md` has an exercise checklist.
Find the first unchecked exercise and start there.
Before you do, spend 5 minutes skimming the CONCEPTS.md to refresh your memory.

---

## Recommended session structure

| Time          | Activity                                                       |
|---------------|----------------------------------------------------------------|
| 0–10 min      | Open yesterday's exercise file. Read your code aloud. Can you  |
|               | explain every line? If not, re-read the relevant CONCEPTS.md   |
|               | section before moving forward.                                 |
| 10–30 min     | Read CONCEPTS.md for today's topic. Run the examples as you go.|
| 30–90+ min    | Work through exercises. Type every single character yourself.  |
|               | This block is the 70%. Protect it.                             |

Total: ~90 minutes. Stretch to 2 hours when you are in flow.

---

## What to do when you are stuck

**First 15 minutes — struggle productively:**
- Re-read the relevant CONCEPTS.md section.
- Run the example file that covers that topic.
- Add `print()` statements to see what your variables actually contain.
- Read the error message carefully — Python error messages are usually accurate.

**After 15 minutes of genuine effort:**
- Open the solution file for that exercise.
- Read it once all the way through.
- **Close the solution file.**
- Retype the solution from memory into your own file.

Do not skip the "close and retype" step. It is the most important part.

**If you are still confused after all that:**
- Search the specific error message on Google or ask on Stack Overflow.
- Python has one of the best beginner communities online.
