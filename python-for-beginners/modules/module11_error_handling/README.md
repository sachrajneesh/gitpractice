# Module 11: Error Handling

> **30/70 reminder** — Error handling is easiest to learn by deliberately breaking
> your existing code and adding try/except around the fragile parts.
> Take the diary app from Module 10 and harden it.

---

## Learning objectives

By the end of this module you will be able to:

- Use `try`/`except` to catch and handle exceptions
- Use `else` (runs if no exception) and `finally` (always runs)
- Catch specific exception types vs catching all exceptions
- Raise your own exceptions with `raise`
- Name the most common built-in exceptions and what causes them

---

## Topics covered

1. `try` / `except` — the basic structure
2. Catching specific exception types: `ValueError`, `TypeError`, `FileNotFoundError`, etc.
3. `else` block — runs only when no exception occurred
4. `finally` block — always runs (cleanup code)
5. `raise` — throwing exceptions deliberately
6. Common built-in exceptions and what triggers them
