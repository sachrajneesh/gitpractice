# Module 14: Mini Projects

> **30/70 reminder** — This module IS the 70%. There is almost no reading.
> Pick a project, read its spec, and build it from scratch.
> These projects integrate everything from Modules 01-13.

---

## Learning objectives

By the end of this module you will have:

- Built three complete, working Python programs from scratch
- Practised planning a program before writing it
- Experienced the full cycle: design → implement → test → refine

---

## Projects

### Project 1 — Number Quiz (Easy) 🟢
A maths quiz game that generates random arithmetic questions, tracks the score,
and reports a final grade at the end. Uses: random, loops, functions, f-strings.

### Project 2 — Expense Tracker (Medium) 🟡
A command-line app to log daily expenses, view totals by category,
and save/load data from a CSV file. Uses: dicts, lists, file I/O, csv module.

### Project 3 — Text-Based Adventure (Hard) 🔴
A small interactive story where the player makes choices that affect the outcome.
Multiple rooms/scenes, inventory system, win/lose conditions.
Uses: dicts, functions, OOP, loops, file I/O (save/load game state).

## Recommended approach

1. Read the project spec carefully.
2. Write a short plan: what data will you need? What functions? What will the main loop do?
3. Build the minimum working version first (just the core loop).
4. Add features one at a time, testing after each addition.
5. Only move to Project 2 once Project 1 works fully.
